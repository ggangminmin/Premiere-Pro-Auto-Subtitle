# 자막 형식 규칙

자동 생성 + 사람이 다듬을 때 모두 지키는 규칙입니다.
(`src/transcribe.py` 에서 1차 적용, **`src/build_srt.py` 출력 직전 단계에서 다시 강제** —
트랜스크립션/번역 워크시트/수기 어느 경로로 들어온 텍스트든 동일하게 보장됩니다.)

## 1. 반복 추임새는 2번까지
- "Oh, oh, oh, oh, oh…" 처럼 같은 추임새가 길게 반복되면 **2회**로 줄입니다. → `Oh oh`
- 대상 예: oh, ah, la, na, woah, yeah, 오, 아, 라 …
- (코드: `transcribe.trim_filler`. SRT 출력 시 `build_srt.write_srt` 가 **ja·ko 양쪽에 다시**
  적용하므로, 번역 칸에 추임새가 길게 들어와도 자동 축약됩니다. `FILLER_WORDS` 에 추가 가능)

## 2. 한 자막에 여러 가사 줄을 합치지 않는다
- 두 가사 줄을 한 자막에 몰아넣지 **않습니다.** 가사 줄이 바뀌면 **다음 큐**로 띄웁니다.
- 1차: `transcribe.split_on_word_gaps` — 단어 사이 공백이 `split_gap`초 이상이면 자동 분할.
  (너무 잘게 쪼개지면 `config.json` 의 `split_gap` 값을 키우세요. 기본 0.8초.)
- 2차(안전망): 노래는 음을 길게 끌어(legato) 두 줄이 한 큐에 붙는 일이 잦습니다. 이때는
  **`ja`·`ko` 양쪽 같은 위치에 ` / ` (공백-슬래시-공백)** 를 넣어 주면, `build_srt.split_merged_lines`
  가 SRT 생성 시 **별도 큐로 자동 분리**하고 시간을 글자수 비례로 나눠 줍니다.
  - 한쪽에만 ` / ` 가 있거나 양쪽 개수가 안 맞으면(짝이 불명확하면) **분리를 보류하고 경고**를
    출력합니다 → 로그를 보고 양쪽에 ` / ` 를 맞춰 넣거나 직접 손봐 주세요.

## 3. 가짜 자막(크레딧/멘트) 제거
- "작사·작곡", "ご視聴ありがとう", "채널 구독" 류의 영상 끝 멘트는 자동 제거합니다.
- (코드: `transcribe.JUNK_RE`)

## 4. 텍스트는 가사 파일이 정답
- AI가 가끔 잘못 알아듣습니다(예: 고유명사). **타이밍은 AI, 텍스트는 가사 파일** 기준으로 교정하세요.
- 워크시트(`_translation.json`)의 `ja` 를 가사 파일과 대조해 고친 뒤 `ko` 를 채우면 됩니다.

## 5. 최소 노출 시간 / 겹침 방지
- 각 자막은 최소 0.6초 이상 노출, 다음 자막과 시간이 겹치지 않게 자동 보정합니다.
- (코드: `build_srt.normalize`)
