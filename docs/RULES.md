# 자막 형식 규칙

자동 생성 + 사람이 다듬을 때 모두 지키는 규칙입니다. (`src/transcribe.py` 에 일부 자동 적용)

## 1. 반복 추임새는 2번까지
- "Oh, oh, oh, oh, oh…" 처럼 같은 추임새가 길게 반복되면 **2회**로 줄입니다. → `Oh oh`
- 대상 예: oh, ah, la, na, woah, yeah, 오, 아, 라 …
- (코드: `transcribe.trim_filler`, `FILLER_WORDS` 에 추가 가능)

## 2. 한 자막에 여러 가사 줄을 합치지 않는다
- 두 가사 줄을 `/` 같은 구분자로 한 자막에 몰아넣지 **않습니다.**
- 가사 줄이 바뀌면 **다음 자막(다음 큐)** 으로 띄웁니다.
- (코드: `transcribe.split_on_word_gaps` — 단어 사이 공백이 `split_gap`초 이상이면 자동 분할.
  너무 잘게 쪼개지면 `config.json` 의 `split_gap` 값을 키우세요. 기본 0.8초.)

## 3. 가짜 자막(크레딧/멘트) 제거
- "작사·작곡", "ご視聴ありがとう", "채널 구독" 류의 영상 끝 멘트는 자동 제거합니다.
- (코드: `transcribe.JUNK_RE`)

## 4. 텍스트는 가사 파일이 정답
- AI가 가끔 잘못 알아듣습니다(예: 고유명사). **타이밍은 AI, 텍스트는 가사 파일** 기준으로 교정하세요.
- 워크시트(`_translation.json`)의 `ja` 를 가사 파일과 대조해 고친 뒤 `ko` 를 채우면 됩니다.

## 5. 최소 노출 시간 / 겹침 방지
- 각 자막은 최소 0.6초 이상 노출, 다음 자막과 시간이 겹치지 않게 자동 보정합니다.
- (코드: `build_srt.normalize`)
